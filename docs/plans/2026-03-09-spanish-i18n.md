# Spanish i18n Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add English/Spanish localization with auto-detection of system language and a UI Language picker in General Settings.

**Architecture:** Use Swift 5.9+ `String(localized:)` with a `Localizable.xcstrings` String Catalog. Language is resolved once at startup (app `init()`) by setting `AppleLanguages` in UserDefaults, which SwiftUI then reads for all `LocalizedStringKey` lookups. Language changes take effect after restart. The project uses `PBXFileSystemSynchronizedRootGroup` so adding the `.xcstrings` file to the folder is enough — no file reference entry needed in the pbxproj.

**Tech Stack:** Swift 5.9+, SwiftUI, Xcode 16 String Catalog (`.xcstrings`)

### Task 2: Apply stored language at app startup

**Files:**
- Modify: `OpenWhisper/OpenWhisper/OpenWhisperApp.swift`

**Step 1: Add `init()` to `OpenWhisperApp` that resolves and applies UI language**

Replace the existing struct opening of `OpenWhisperApp` with:

```swift
@main
struct OpenWhisperApp: App {
    @State private var appState = AppState()
    @State private var updaterService = UpdaterService()
    @State private var didAutoSetup = false

    init() {
        Self.applyStoredLanguage()
    }

    // Supported UI languages beyond English
    private static let supportedUICodes: Set<String> = ["es"]

    private static func applyStoredLanguage() {
        let stored = UserDefaults.standard.string(forKey: Constants.Defaults.uiLanguage) ?? "auto"
        let code: String
        if stored == "auto" {
            let systemCode = Locale.current.language.languageCode?.identifier ?? "en"
            code = supportedUICodes.contains(systemCode) ? systemCode : "en"
        } else {
            code = stored
        }
        UserDefaults.standard.set([code], forKey: "AppleLanguages")
        UserDefaults.standard.synchronize()
    }
    // ... rest of body unchanged
```

### Task 3: Add UI Language picker to GeneralSettingsTab

**Files:**
- Modify: `OpenWhisper/OpenWhisper/Views/SettingsView.swift`

**Step 1: Add language picker and restart notice to `GeneralSettingsTab.body`**

In `GeneralSettingsTab`, find the first `Section {` block (the one with `Toggle("Launch at Login", ...)` and `Picker("Language", ...)`). Add a new section **after** this section and before `Section("Output")`:

```swift
Section {
    Toggle(String(localized: "Launch at Login"), isOn: $settings.launchAtLogin)

    Picker(String(localized: "Language"), selection: $settings.selectedLanguage) {
        ForEach(Constants.SupportedLanguages.all, id: \.code) { lang in
            Text(lang.name).tag(lang.code)
        }
    }

    Picker(String(localized: "App Language"), selection: $settings.uiLanguage) {
        Text(String(localized: "Auto")).tag("auto")
        Text("English").tag("en")
        Text("Español").tag("es")
    }

    if settings.uiLanguage != originalUILanguage {
        Text(String(localized: "Restart required to apply language change."))
            .font(.caption)
            .foregroundStyle(.orange)
    }
}
```

Add `@State private var originalUILanguage: String = UserDefaults.standard.string(forKey: Constants.Defaults.uiLanguage) ?? "auto"` as a property of `GeneralSettingsTab`.

**Note:** `originalUILanguage` stores the language at view appearance so the restart notice only shows when the value differs from what was applied at launch.

**Step 2: Commit**
```bash

git commit -m "feat: add App Language picker to General settings tab"
```
